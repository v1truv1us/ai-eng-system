# Plan: Make thermo-nuclear review skills model-invokable + workflow-ready

## Problem
The four `thermo-nuclear-*` skills are not usable by the model and don't function inside review workflows. The user wants them model-invoked so they can run together in a workflow.

## Root cause (confirmed)
All four skills are misclassified in their frontmatter:

```yaml
metadata:
  category: user-invoked      # WRONG for these descriptions
disable-model-invocation: true # blocks model/agent invocation
```

Per the repo's own `sync-skill-taxonomy` rules (`sync-skill-taxonomy/SKILL.md:30-39`), a skill is **user-invoked** only if its description contains explicit-trigger language ("Use only when the user...", "explicitly types"). The thermo-nuclear descriptions use capability language ("Use for a thermo-nuclear ... review"), so they must be **model-invoked**. The current flag prevents the model from auto-invoking them and prevents the `deep-review` workflow's agents from loading them.

Note: the skills ARE deployed to `.opencode/skill/thermo-nuclear-*/SKILL.md` (the earlier "not available" symptom was the `disable-model-invocation` flag, not missing files). Opencode does not read repo-root `skills/` directly — `bun run build` syncs `skills/` → `.opencode/skill/`, `.claude/skills/`, `dist/`.

## Key finding: the workflow already exists
`content/commands/deep-review.md:40-43` already orchestrates all four skills as a parallel workflow via `/ai-eng/deep-review`:

| Axis | Agent | Skill |
|------|-------|-------|
| Code Quality | code-reviewer | thermo-nuclear-code-quality-review |
| Security | security-scanner | thermo-nuclear-security-review |
| Architecture | architect-advisor | thermo-nuclear-architecture-review |
| Performance | performance-engineer | thermo-nuclear-performance-review |

No new command is required. The fix is purely reclassifying the skills so the workflow can actually invoke them.

## Changes

### 1. Reclassify the 4 source skills (canonical source in `skills/`)
For each file, change `metadata.category: user-invoked` → `model-invoked` AND delete the `disable-model-invocation: true` line:

- `skills/thermo-nuclear-architecture-review/SKILL.md:4-6`
- `skills/thermo-nuclear-code-quality-review/SKILL.md:4-6`
- `skills/thermo-nuclear-performance-review/SKILL.md:4-6`
- `skills/thermo-nuclear-security-review/SKILL.md:4-6`

Target frontmatter for each:
```yaml

  category: model-invoked
```
(no `disable-model-invocation` line)

`bun run build` runs `scripts/build-all.ts`, which calls `buildOpenCode()` → `copySkillsPreservePath()` (`build.ts:659`) to repopulate `.opencode/skill/thermo-nuclear-*/SKILL.md` with the corrected frontmatter.
