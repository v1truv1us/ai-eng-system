# Plan: Fix OpenCode Agent `tools` Format and Clarify agent-skills Relationship

## Diagnosis

The error at `~/.config/opencode/agent/ai-eng/general/claude-refactor-agent.md` is caused by a **stale installed file** with a Claude Codeâ€“style `tools` array, which OpenCode rejects (it expects `tools` as an object of booleans).

- The current `build.ts` already transforms array â†’ object for OpenCode output (`build.ts:365-378`), so generated `dist/.opencode/` files are valid.
- The root cause is that the **canonical source** `content/agents/claude-*.md` uses array syntax, while the repo's own format guide (`skills/plugin-dev/references/agent-format.md`) says canonical agents should use object syntax.
- Other agents (`code-reviewer.md`, `full-stack-developer.md`, etc.) already follow the object format.

**agent-skills is not a replacement for the CLI.** It is a skill format/specification that this repo already implements (`scripts/lib/agent-skills.ts`, `.agents/skills/` install path). The CLI still provides multi-platform packaging, commands, agents, install/clean workflow, and the `ralph` runner that agent-skills does not provide. The simplification path is to rely on the Agent Skills convention more heavily where it reduces duplication, not to remove the CLI.

## Implementation Steps

### 1. Convert canonical `claude-*.md` agents to object `tools`

Update these six files so their `tools` frontmatter uses object syntax:

- `content/agents/claude-conductor.md`
- `content/agents/claude-debugger-agent.md`
- `content/agents/claude-lookup-agent.md`
- `content/agents/claude-planner-agent.md`
- `content/agents/claude-refactor-agent.md`
- `content/agents/claude-work-agent.md`

Example conversion for `claude-refactor-agent.md`:

```yaml
# Before

  - Read
  - Write
  - Edit
  - Grep
  - Glob
  - Bash

# After

  read: true
  write: true
  edit: true
  grep: true
  glob: true
  bash: true
```

Leave `name`, `description`, `mode`, `model`, and `temperature` unchanged.

### 2. Add a Claude Code output transform in `build.ts`

Currently `buildClaude()` copies canonical agents verbatim (`build.ts:490-543`). Once canonical sources become objects, Claude Code output would also be objects, which is invalid for Claude. Add a transform that converts object `tools` back to PascalCase arrays for Claude output.

- Add `transformAgentMarkdownForClaude()` near `transformAgentMarkdownForOpenCode()` (around `build.ts:404`).
- Convert object tools to array: `read: true` â†’ `Read`, etc.
- Strip `permission` from Claude output (Claude does not support it).
- Update `buildClaude()` to use the transform instead of copying verbatim.
- Keep `transformAgentMarkdownForOpenCode()` as a defensive fallback for legacy arrays, but update its comment to note that canonical sources are now validated to be objects.

### 3. Add canonical-source validation in `build.ts`

In `validateContentOnly()` (`build.ts:1990`), inside the `agentFiles` loop after the `mode` check, add:

```ts
if (meta.tools) {
    if (Array.isArray(meta.tools)) {
        errors.push(`${fp}: tools must be an OpenCode object, not a Claude Code array`);
    } else if (typeof meta.tools !== "object" || meta.tools === null) {
        errors.push(`${fp}: tools must be an object`);
    } else {
        for (const [tool, value] of Object.entries(meta.tools)) {
            if (typeof value !== "boolean") {
                errors.push(`${fp}: tools.${tool} must be a boolean value`);
            }
        }
    }
}
```

### 4. Add post-build output validation in `validateAgents()`
