# Plan: Add Graphify Skill for Graph/LLM Applications

## Overview

The Instagram post from @latent_space shows a "Graphify Skill" framework for building Graph + LLM applications. Currently, there's a **significant gap** - the ai-eng-system has AI/LLM agents but **no dedicated skill for knowledge graph development**.

## What's Shown in Instagram Post

The "Graphify Skill" covers 6 categories:

| Category | Contents |
|----------|---------|
| **Graph Storage** | memory, postgres, neo4j, redis, aws-neptune |
| **Graph Manipulation** | traversal, shortest path, centrality, community detection, page rank |
| **Graph Extraction** | pdfs, code, websites, json/yaml, sql, docs |
| **Graph Construction** | entity extraction, relationship extraction, semantic search, embeddings |
| **LLM Graph Integration** | graph RAG, graph tools, tool creation, subagent orchestration |
| **Graph Viz** | d3, mermaid, visjs, graphviz |

## Gap Analysis

**Existing AI/LLM capabilities in ai-eng-system:**
- `ai-engineer` - LLM application development
- `ml-engineer` - ML model development  
- `prompt-optimizer` - Prompt enhancement
- `agent-developer` - MCP, tool calling, multi-agent orchestration

**Missing (not covered by any existing skill):**
- Knowledge graph storage selection (which DB to use when)
- Graph algorithms (when to use traversal vs shortest path vs centrality)
- Extracting graphs from documents, code, websites
- LLM-based entity/relationship extraction
- Graph RAG patterns
- Graph visualization

## Recommended Implementation

Create a new skill at `.opencode/skill/graphify/SKILL.md`:

### Core Sections

1. **Graph Storage Selection**
   - When to use in-memory (prototyping)
   - When to use PostgreSQL with extensions (relational data)
   - When to use Neo4j (complex relationship queries)
   - When to use Redis (caching, real-time)
   - When to use AWS Neptune (managed graph)

2. **Graph Algorithms by Use Case**
   - Path finding: BFS/DFS for exploration, Dijkstra for weighted routing
   - Centrality: PageRank for importance, Betweenness for bridges
   - Community detection: Louvain for clustering
   - Recommendations: Collaborative filtering on graphs

3. **Graph Extraction Sources**
   - From documents (PDFs, Markdown) → document chunking + entity extraction
   - From code (AST parsing) → dependency graphs
   - From websites → link graphs
   - From SQL → schema graphs
   - From JSON/YAML → configuration graphs

4. **LLM Graph Construction**
   - Entity extraction prompts (what entities, what relationships)
   - Relationship confidence scoring
   - Embedding for semantic search
   - Incremental graph building

5. **Graph RAG Patterns**
   - Graph as knowledge base for context
   - Graph tool definitions for agentic use
   - Hybrid: vector + graph retrieval

6. **Graph Visualization**
   - Mermaid for code documentation
   - D3.js for web apps
   - Graphviz for static diagrams

## Critical Files to Create
