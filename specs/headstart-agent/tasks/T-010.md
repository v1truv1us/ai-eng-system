# T-010: Implement Monthly Audit Task

**Depends On:** T-005, T-006, T-007
**Priority:** high
**Estimated Time:** 75 min
**Complexity:** High

## Acceptance Criteria
- [ ] `src/tasks/monthly-audit.ts` exports a `Task` implementation with `type = 'monthly-audit'`
- [ ] `run()` calls `ensureRepo()` to get fresh repo
- [ ] Runs `bun outdated` and parses output to list outdated packages
- [ ] Runs `bun audit` and captures vulnerability count
- [ ] Runs Lighthouse (mobile + desktop) and extracts scores for Performance, Accessibility, Best Practices, SEO
- [ ] Checks content freshness: last newsletter date, copyright year, sample of external links
- [ ] If actionable changes exist (outdated deps, Lighthouse issues, content stale):
  - Updates minor/patch dependencies in `package.json`
  - Creates branch `agent/monthly-{YYYY-MM-DD}`
  - Commits with message `Monthly audit: update dependencies ({date})`
  - Pushes branch and opens PR with audit report body
- [ ] If no changes needed: returns `success=true` with "no action required" message
- [ ] Unit tests pass with mocked command outputs: deps outdated, no changes, Lighthouse parsing
