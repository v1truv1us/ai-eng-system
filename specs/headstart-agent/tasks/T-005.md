# T-005: Implement SQLite Job Queue

**Depends On:** T-002, T-004
**Priority:** critical
**Estimated Time:** 45 min
**Complexity:** Medium

## Acceptance Criteria
- [ ] `src/queue.ts` exports functions: `createJob`, `claimJob`, `completeJob`, `failJob`, `getJob`, `listJobs`
- [ ] SQLite schema matches spec: `jobs` table with all fields (id, type, status, payload, branch, prUrl, error, createdAt, startedAt, completedAt, retryCount)
- [ ] `createJob` generates UUID, sets status to `pending`, computes idempotency key from payload hash
- [ ] `claimJob` finds oldest `pending` job, sets `status=running`, updates `startedAt`
- [ ] `completeJob` sets `status=completed`, stores `branch` and `prUrl`, updates `completedAt`
- [ ] `failJob` increments `retryCount`; if `retryCount < 3`, requeues with `status=pending`; else `status=failed`
- [ ] `listJobs` supports optional `status` filter and `limit` parameter
- [ ] Database path uses `config.DATA_DIR` (default `/data`)
- [ ] Unit tests pass: create, claim, complete, fail-with-retry, max-retries-exceeded, idempotency
