# T-008: Implement Webhook Server

**Depends On:** T-003, T-005
**Priority:** critical
**Estimated Time:** 50 min
**Complexity:** Medium

## Acceptance Criteria
- [ ] `src/server.ts` exports a Fastify instance with routes: `POST /webhook/email`, `GET /health`, `GET /jobs`
- [ ] `POST /webhook/email` accepts multipart/form-data with `sender`, `subject`, `attachment`
- [ ] Validates sender against `config.ALLOWED_SENDERS` (case-insensitive); returns 403 if unauthorized
- [ ] Validates exactly one PDF attachment; returns 400 if missing or multiple
- [ ] Validates subject matches `/\w+ Newsletter \d{4}/i`; returns 400 if malformed
- [ ] Verifies Mailgun webhook signature using `config.MAILGUN_API_KEY` and `config.WEBHOOK_SECRET`
- [ ] On valid request: saves PDF to temp path, creates queue job with `type='newsletter'`, returns 202 with `jobId`
- [ ] `GET /health` returns JSON with `status`, `version`, and queue counts
- [ ] `GET /jobs` supports `?limit=` and `?status=` query params
- [ ] Integration tests pass: valid → 202, bad sender → 403, no attachment → 400, bad subject → 400

## Files
- `src/server.ts` — Create Fastify server with all routes
- `tests/integration/webhook.test.ts` — Create integration tests
