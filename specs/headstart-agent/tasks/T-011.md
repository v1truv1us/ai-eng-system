# T-011: Implement Alerting (Sentry + Email)

**Depends On:** T-002, T-003
**Priority:** high
**Estimated Time:** 40 min
**Complexity:** Medium

## Acceptance Criteria
- [ ] `src/alerts.ts` exports: `initSentry()`, `reportError(error, context)`, `sendAlertEmail(subject, body)`
- [ ] `initSentry()` initializes `@sentry/node` with `config.SENTRY_DSN` if provided; no-op if missing
- [ ] `reportError()` captures exception in Sentry with tags: `jobId`, `jobType`, `retryCount`
- [ ] `sendAlertEmail()` uses nodemailer with SMTP config from env vars; no-op if SMTP not configured
- [ ] Alert email subject: `[Headstart Agent] Job Failed: {jobType} ({jobId})`
- [ ] Alert email body includes: job ID, type, payload, error message, timestamp
- [ ] Unit tests pass: Sentry captures error with context, email sends with correct content, no-op when config missing

## Files
- `src/alerts.ts` — Create alerting module
- `tests/unit/alerts.test.ts` — Create unit tests with mocked Sentry and nodemailer
