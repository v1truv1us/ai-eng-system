## How to complete this table

Each row's pairs are in `benchmarks/results/<skill>/`. To judge them:

```bash
# Requires a reachable OpenCode server (the G-Eval LLM-as-judge).
bun run eval:run            # judges all pairs under benchmarks/results/
# or per skill:
bun run eval:skill <name>   # re-emits pairs and judges them in one pass
```

For each skill, fill the columns from its `_eval.json` files (overall.winner):
- **Enhanced > Baseline** — the skill earns its place. KEEP.
- **Tied** — retest with harder cases; lean RETIRE if content is public knowledge.
- **Baseline ≥ Enhanced** — RETIRE (Batch 2e). The skill is fighting the model.

Re-run this table on every model upgrade (RETEST skills are shelf-life-bound).
